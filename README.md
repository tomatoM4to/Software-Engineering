# 다중 AI 에이전트 기반 장/단기 스윙 퀀트 트레이딩 시스템

## 1. Vision statement
> 데이터 기반의 알고리즘 매매 엔진과 LLM 기반 다중 AI 페르소나를 결합한 고신뢰성 퀀트 트레이딩 API 서버 구축

본 프로젝트는 한국투자증권(KIS) API 를 통한 장기/단기 스윙 전략(기술적 분석), 퀀트 지표, 그리고 네이버 뉴스 API 를 통한 다중 AI Agent 를 결합하여 높은 신뢰성을 갖춘 트레이팅 시스템을 구축하는 것을 목표로 합니다.

## 2. Project goals & Scope
### Project Goals
1. **비동기 데이터 파이프라인 및 정규화**: 증권사 및 외부 API를 활용해 시세 데이터(일봉/분봉), 재무 데이터, 텍스트(뉴스) 데이터를 수집하고 SQLite에 최적화하여 적재하는 백그라운드 스케줄링 시스템 구축.
2. **기술적 분석 알고리즘 구축**: Pandas, NumPy, TA-Lib 등을 활용하여 장기 스윙(일봉)과 단기 스윙(분봉) 전략에 필요한 기술적 지표 및 필터링 로직을 구현하는 API 엔드포인트 개발.
3. **퀀트 지표 및 뉴스 데이터 통합**: 수집된 시세 데이터와 뉴스 데이터를 LLM 프롬프트 엔지니어링을 통해 통합 분석하여, 종목별 투자 의견과 리스크 평가를 제공하는 시스템 구축.
4. **다중 AI 에이전트 설계 및 통합**: 서로 다른 투자 성향(보수적, 공격적)을 가진 다중 AI 페르소나를 설계하여, 동일한 데이터에 대해 다양한 시각에서 분석하고 종합하는 시스템 구축.

### Project Scope
* 데이터 수집 계층
    - KIS API 연동을 통한 시세 데이터(일봉/분봉) 및 재무 데이터 수집.
    - 네이버 뉴스 API 연동을 통한 종목별 뉴스 텍스트 데이터 수집.
    - 수집된 데이터를 SQLite에 최적화하여 적재하는 백그라운드 스케줄링 시스템 구축.
* 분석 계층
    - Pandas, NumPy, TA-Lib 등을 활용하여 장기 스윙(일봉)과 단기 스윙(분봉) 전략에 필요한 기술적 지표 및 필터링 로직 구현.
* Multi AI Agent 계층
    - LLM 프롬프트 엔지니어링을 통해 서로 다른 투자 성향(보수적, 공격적)을 가진 다중 AI 페르소나 설계.
    - 동일한 데이터에 대해 다양한 시각에서 분석하고 종합하는 시스템 구축.

## 3. Stakeholders & Users
* Stakeholders
    - 클라이언트 (투자 전략가): 스윙 매매 로직(AND/OR 조건)의 도메인 지식을 제공하며, AI 에이전트의 분석 결과가 실제 트레이딩에 유효한지 검증하고 피드백을 제공하는 역할.
    - 개발팀: 아키텍처 설계, API 연동, 다중 AI 프롬프트 엔지니어링, 인프라 구축 등을 담당
* Users
    - 단순한 차트 검색기를 넘어, 종목의 뉴스와 시장 심리까지 AI의 객관적인 시선으로 점검받고 싶은 개인 트레이더.
    - 기술적 분석(차트)과 기본적 분석(뉴스)을 융합하여 안전한 장/단기 스윙 투자를 지향하는 직장인 투자자.

## 4. Milestone
본 프로젝트는 Agile 방법론을 채택하여 핵심 기능부터 점진적으로 확장하는 4단계 로 진행됩니다.

1. Phase 1: 데이터 인프라 구축 (1개월)
    - 시스템 아키텍처 및 SQLite DB 스키마(ERD) 설계.
    - KIS API 및 네이버 뉴스 API 연동 모듈 개발.
    - 비동기 백그라운드 스케줄러 구축 및 데이터 적재 테스트 완료.
2. Phase 2: 기술적 분석 알고리즘 개발 (0.5개월)
    - Pandas, TA-Lib 기반 기술적 지표(장/단기 스윙) 연산 코어 모듈 구현.
3. Phase 3: 퀀트 지표 및 뉴스 데이터 통합 (0.5개월)
    - 시세/뉴스 데이터 전처리(NLP) 파이프라인 구축.
    - 단일 AI 모델 대상 프롬프트 최적화 및 종목별 투자 의견/리스크 평가 로직 연동.
4. Phase 4: 다중 AI 에이전트 설계 및 통합 (0.5개월)
    - 보수적/공격적 페르소나 분리 및 다중 에이전트 라우팅(Routing) 구조 구현.
    - 에이전트 간 합의 알고리즘 설계
    - 예외 처리, Unit Test 진행 및 최종 산출물 정리.
